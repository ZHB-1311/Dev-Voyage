## 2.1 KNN：最近的邻居

### 2.1.1 算法直觉

KNN 的核心思想只有一句话：**"长得像，就是一类。"**

```
测试图像
    │
    ▼
┌─────────────────────────────┐
│  计算与所有训练图像的距离     │
│                             │
│  距离：d₁, d₂, d₃, ..., dₙ  │
│       ↓                     │
│  找到最近的 k 个邻居         │
│       ↓                     │
│  多数投票决定类别            │
└─────────────────────────────┘
```

KNN 没有任何"训练"——它是一个**懒惰学习器 (Lazy Learner)**。它把所有计算推迟到预测时。相比之下，神经网络和决策树是**急切学习器 (Eager Learner)**——在训练阶段就构建了内部表示。

### 2.1.2 距离度量

距离的选择直接影响 KNN 的行为。三种常用距离：

| 距离 | 公式 | 几何含义 | 适用场景 |
|------|------|----------|----------|
| **L1（曼哈顿）** | $\sum \vert a_i - b_i \vert$ | 街区距离 | 高维稀疏数据 |
| **L2（欧几里得）** | $\sqrt{\sum (a_i - b_i)^2}$ | 直线距离 | 通用、连续特征 |
| **余弦相似度** | $\frac{a \cdot b}{\|a\|\|b\|}$ | 夹角余弦 | 文本、高维稠密向量 |

::: info 数学知识
**L2 范数（欧几里得距离）**

**定义：** 向量各分量平方和的平方根，衡量向量的"长度"。

**公式：**
$$
\|\mathbf{x}\|_2 = \sqrt{\sum_{i=1}^{n} x_i^2}
$$

**两点之间的距离：**
$$
d(\mathbf{a}, \mathbf{b}) = \|\mathbf{a} - \mathbf{b}\|_2 = \sqrt{\sum_{i=1}^{n} (a_i - b_i)^2}
$$

**其他范数：**
- L1 范数（曼哈顿距离）：$\|\mathbf{x}\|_1 = \sum_{i=1}^{n} |x_i|$——适用于稀疏向量
- L∞ 范数：$\|\mathbf{x}\|_\infty = \max_i |x_i|$——只关心最大的分量差

**在 AI 中的应用：** KNN 核心度量、正则化（L1/L2 Regularization）、VAE 的重建损失。
:::

### 2.1.3 从零实现

```python
import numpy as np
from collections import Counter

class KNN:
    """最近邻分类器——懒惰学习的极端"""

    def __init__(self, k=3, metric='l2'):
        self.k = k
        self.metric = metric

    def fit(self, X, y):
        """训练：只是记下来。什么都不算。"""
        self.X_train = X
        self.y_train = y

    def predict(self, X):
        """预测：对每个待测样本，找最近的 k 个邻居，投票。"""
        return np.array([self._predict_one(x) for x in X])

    def _predict_one(self, x):
        # 计算到所有训练样本的距离
        if self.metric == 'l1':
            distances = np.sum(np.abs(self.X_train - x), axis=1)
        else:  # l2
            distances = np.sqrt(np.sum((self.X_train - x) ** 2, axis=1))

        # 找 k 个最近邻的索引
        k_indices = np.argsort(distances)[:self.k]
        k_labels = self.y_train[k_indices]

        # 多数投票
        return Counter(k_labels).most_common(1)[0][0]
```

`fit()` 什么都没算——这就是懒惰学习。`predict()` 遍历所有训练样本、计算距离、排序、投票。

### 2.1.4 k 值的影响

k 是 KNN 唯一的超参数。它的选择直接决定了模型的决策边界：

```
k=1：            k=3：            k=15：
  · · ●            · ○ ●            · ○ ●
  · ● ●            · ○ ●            · ○ ●
  ○ ○ ●            ○ ○ ●            ○ ○ ○

每个决策边界       边界开始平滑       边界更平滑但细节丢失
锯齿状——过拟合
```

- **k=1**：每个训练样本就是它自己的岛。决策边界紧贴数据点——过拟合的经典表现。
- **k=N（全部样本）**：不管输入是什么，都预测训练集中最多的类别——欠拟合。
- **合适的 k**：在两者之间，通常通过交叉验证选择。

::: tip 场景
「k=1。训练集 100%。」我把结果投到显示器上。「本天才——」

红莉栖拿起键盘。测试集。82%。

「k=1 的时候，你那个数字 3 的测试样本——它的最近邻是一个训练集里的数字 8。因为那个 8 写得跟 3 一样潦草。你的模型把一个看起来像 3 的 8 当成了 3。」

「把 k 调到 5。它看了看另外 4 个邻居——三个是 8。改口了。」
:::

### 2.1.5 维度灾难

KNN 最大的敌人不是算法本身，而是**维度**。

::: info 数学知识
**维度灾难 (Curse of Dimensionality)**

在高维空间中，距离失去意义——所有点之间的距离趋向于相等。

直觉理解：考虑单位超立方体 $[0,1]^d$。在 $d$ 维空间中，随机两个点的期望欧几里得距离近似为：

$$
\mathbb{E}[d(\mathbf{x}, \mathbf{y})] \approx \sqrt{\frac{d}{6}}
$$

当 $d=784$（MNIST 的维度）：$\sqrt{784/6} \approx 11.4$

而超立方体的对角长度为 $\sqrt{d} \approx 28$。两个随机点之间的距离是 11.4——基本上是"中等距离"。在高维空间中，没有"近的邻居"和"远的陌生人"之分——大家都是差不多的距离。

**后果：** KNN 在高维数据上几乎无法工作。你要么降维（PCA），要么换一个不依赖距离的模型。

**在 AI 中的应用：** 理解为什么深度学习需要层级特征提取——越往高层，维度越低，语义越浓缩。
:::

::: tip 冷知识
KNN 不是"近几年的发明"。Fix 和 Hodges 在 1951 年就提出了非参数分类规则的原型——比感知机（1957）还早 6 年，比 k-d 树（1975）早 24 年。但当时没人有 60000 个样本的数据集。一个算法诞生后等了 50 年，等到了 MNIST，等到了足够的计算力。就像中世纪画好了火箭图纸，等到了 20 世纪才有了燃料。
:::

**KNN 的完整画像：**

| 优点 | 缺点 |
|------|------|
| 零训练时间 | 预测慢（遍历所有训练样本） |
| 可解释（"为什么判为 3？因为最近的三个邻居都是 3"） | 内存大（存储全部训练数据） |
| 无假设（不假设数据服从某种分布） | 维度灾难（高维中距离失去意义） |
| 天然支持多分类 | 对噪声敏感 |

## 2.2 决策树：分而治之

### 2.2.1 算法直觉

KNN 的致命弱点是预测速度。每次预测都要遍历六万个邻居。

决策树的回答是：**不需要看六万个邻居。只需要问对 20 个问题。**

```
输入图像 (784 像素)
    │
    ▼
像素 156 的值 > 128 吗？
    ├── 是 → 像素 382 的值 > 64 吗？
    │        ├── 是 → 像素 45 的值 > 200 吗？
    │        │        ├── 是 → 【数字 8】
    │        │        └── 否 → 【数字 3】
    │        └── 否 → 像素 201 的值 > 90 吗？
    │                 ├── 是 → 【数字 0】
    │                 └── 否 → 【数字 5】
    └── 否 → 像素 73 的值 > 180 吗？
             ├── 是 → 【数字 9】
             └── 否 → 【数字 1】
```

每一步问一个"是否"问题——把数据切成两半。问完一串之后，你不需要看六个万个邻居。你只需要看 20 个像素。

### 2.2.2 分裂标准：把数据变"纯"

构建决策树的核心问题是：**每个节点选哪个特征、哪个阈值来分裂，能把数据最干净地切成两半？**

"干净"需要量化——这就要用到信息论。

::: info 数学知识
**信息熵 (Information Entropy)**

**定义：** 香农熵衡量一个随机变量的不确定性。分布越均匀，熵越大；分布越集中，熵越小。

**公式：**
$$
H(S) = -\sum_{c=1}^{C} p_c \log_2 p_c
$$

其中 $p_c$ 是类别 $c$ 在集合 $S$ 中的比例，$C$ 是类别总数。

**特殊情况：**
- 所有样本是同一类（$p_1=1$, 其余为 0）：$H = 0$（完全确定——零不确定性）
- 各类均匀分布（$p_c = 1/C$）：$H = \log_2 C$（最大不确定性）

**信息增益 (Information Gain)：**

分裂前的熵减去分裂后加权平均的熵：
$$
IG(S, A) = H(S) - \sum_{v \in \text{Values}(A)} \frac{|S_v|}{|S|} H(S_v)
$$

其中 $A$ 是候选分裂特征，$S_v$ 是分裂后特征 $A$ 取值为 $v$ 的子集。

**在 AI 中的应用：** 决策树的分裂标准、特征选择的评分函数。
:::

::: tip 冷知识
信息熵用的是 Claude Shannon 1948 年《通信的数学理论》里的定义——同一篇论文奠定了整个信息论学科。Shannon 用 $\log_2$（以 2 为底的对数）来算一条电报线路能传多少 bit 的信息。70 年后，你用同一个公式来决定决策树该在哪个像素值上分裂。同一个数学工具，从电报线到决策树——信息论是 AI 最底层的语言之一，Ch8 的交叉熵损失、Ch11 的注意力缩放因子都会回到这里。
:::

**基尼不纯度 (Gini Impurity)：** 另一种衡量"不纯度"的方式，CART 决策树的默认选择：

$$
Gini(S) = 1 - \sum_{c=1}^{C} p_c^2
$$

当所有样本属于同一类时，$Gini = 0$（纯）。当各类均匀分布时，$Gini$ 达到最大值 $1 - 1/C$。

基尼不纯度的计算比熵更快（没有对数运算），实际效果通常与信息增益相差无几。大多数决策树实现（包括 scikit-learn）默认使用基尼不纯度。

::: tip 冷知识
基尼不纯度的命名者 Corrado Gini 是意大利统计学家。他最出名的不是这个——是衡量收入不平等的**基尼系数**。同一个数学直觉：基尼系数衡量一个社会的财富分布有多不均匀，基尼不纯度衡量一堆数据的类别分布有多不纯。桶子说："统计学家都是极端的人——不是研究贫富差距就是研究数据纯度喵。"
:::

### 2.2.3 从零实现

```python
import numpy as np

class DecisionTree:
    """决策树分类器——CART 风格"""

    def __init__(self, max_depth=None, min_samples_split=2):
        self.max_depth = max_depth
        self.min_samples_split = min_samples_split
        self.tree = None

    def fit(self, X, y):
        self.tree = self._build_tree(X, y, depth=0)
        return self

    def _gini(self, y):
        """计算基尼不纯度"""
        _, counts = np.unique(y, return_counts=True)
        probs = counts / len(y)
        return 1 - np.sum(probs ** 2)

    def _best_split(self, X, y):
        """遍历所有特征和阈值，找到最佳分裂点"""
        best_gain = -1
        best_feat, best_thresh = None, None
        current_gini = self._gini(y)

        n_samples, n_features = X.shape

        for feat in range(n_features):
            thresholds = np.unique(X[:, feat])
            for thresh in thresholds:
                left_mask = X[:, feat] <= thresh
                right_mask = ~left_mask

                if left_mask.sum() == 0 or right_mask.sum() == 0:
                    continue

                # 加权平均基尼不纯度
                left_gini = self._gini(y[left_mask])
                right_gini = self._gini(y[right_mask])
                n_left, n_right = left_mask.sum(), right_mask.sum()
                weighted_gini = (n_left * left_gini + n_right * right_gini) / n_samples

                gain = current_gini - weighted_gini
                if gain > best_gain:
                    best_gain = gain
                    best_feat = feat
                    best_thresh = thresh

        return best_feat, best_thresh, best_gain

    def _build_tree(self, X, y, depth):
        n_samples = len(y)

        # 停止条件：达到最大深度 / 样本太少 / 已经纯了
        if (self.max_depth and depth >= self.max_depth) or \
           n_samples < self.min_samples_split or \
           len(np.unique(y)) == 1:
            return {'label': np.bincount(y).argmax()}

        feat, thresh, gain = self._best_split(X, y)

        # 如果找不到有效分裂（所有特征都分不开）
        if feat is None:
            return {'label': np.bincount(y).argmax()}

        left_mask = X[:, feat] <= thresh
        right_mask = ~left_mask

        return {
            'feature': feat,
            'threshold': thresh,
            'left': self._build_tree(X[left_mask], y[left_mask], depth + 1),
            'right': self._build_tree(X[right_mask], y[right_mask], depth + 1),
        }

    def _predict_one(self, x, node):
        if 'label' in node:
            return node['label']
        if x[node['feature']] <= node['threshold']:
            return self._predict_one(x, node['left'])
        else:
            return self._predict_one(x, node['right'])

    def predict(self, X):
        return np.array([self._predict_one(x, self.tree) for x in X])
```

「这个模型——我用肉眼完全能看懂它在干什么。每一步都有理由。每一步都是一个像素和一个阈值。」

「因为它不是黑盒。」她说。「决策树是唯一一个你可以完全追踪推理过程的模型。其他人用 SHAP 和 LIME 来强行解释黑盒。决策树不需要。它天生就是透明的。」

### 2.2.4 过拟合与剪枝

决策树有一个致命弱点：它太容易过拟合。如果不加限制，它会一直分裂，直到每个叶子节点只有一个样本——

```
深度 0: 基尼 = 0.90 (10 类混合)
深度 1: 基尼 = 0.72
深度 5: 基尼 = 0.35
...
深度 20: 基尼 = 0.00 —— 每个叶子都是纯的，每个叶子只有 1 个样本
训练准确率 = 100%，测试准确率 = 58%
```

两种策略防止过拟合：

| 策略 | 做法 | 优缺点 |
|------|------|--------|
| **预剪枝 (Pre-pruning)** | 限制 max_depth、min_samples_split、min_samples_leaf | 简单直接，但可能过于保守 |
| **后剪枝 (Post-pruning)** | 先长满整棵树，再自底向上剪掉"贡献小"的分支 | 更精确，但计算量更大 |

实践中，预剪枝通常已经足够——设置合理的 `max_depth` 和 `min_samples_split` 就能在大多数场景下达到好的泛化效果。

## 2.3 集成学习：群体的智慧

### 2.3.1 核心直觉

一棵决策树容易过拟合。但如果——你有 100 棵决策树，每棵看到的只是数据的一小部分，它们集体投票？

```
单棵决策树：
  训练集 100% → 测试集 58%
  ↑ 记住了噪声

100 棵树投票：
  每棵训练集 ~80% → 测试集 95%
  ↑ 每棵都有偏好，但偏的方向各不相同
  ↑ 投票后，偏好的噪声互相抵消，真正的信号留下来
```

这就是集成学习的核心洞见：**一群弱的投票者可以胜过单个强的专家，前提是它们犯的错不相关。**

### 2.3.2 Bagging 与随机森林

**Bagging (Bootstrap Aggregating)**：

1. 从原始训练集中，**有放回**地抽取 N 个 Bootstrap 样本（每个样本大小和原始集相同）
2. 在每个 Bootstrap 样本上训练一个模型
3. 预测时，所有模型投票（分类）或取平均（回归）

每个 Bootstrap 样本大约包含原始数据 63.2% 的样本（$1 - 1/e \approx 0.632$），剩下 36.8% 的样本从未被选中——这被称为**袋外样本 (Out-of-Bag, OOB)**，可以用作免费的验证集。

**随机森林 = Bagging + 特征随机采样**：

不只是数据随机——连特征也随机。每次分裂时，只从随机选出的 $\sqrt{n\_features}$ 个特征中选择最佳分裂。两个层次的随机性叠加：

```
Bagging： 每棵树看到约 63% 的样本（随机）
      +
随机森林：每次分裂只考虑 √n_features 个特征（随机）
      ↓
树与树之间尽量"不一样" → 集体投票更稳定
```

```python
import numpy as np

class RandomForest:
    """随机森林分类器"""

    def __init__(self, n_trees=100, max_depth=None, max_features='sqrt'):
        self.n_trees = n_trees
        self.max_depth = max_depth
        self.max_features = max_features
        self.trees = []

    def fit(self, X, y):
        n_samples, n_features = X.shape
        self.n_features_sampled = int(np.sqrt(n_features)) if self.max_features == 'sqrt' else n_features

        for _ in range(self.n_trees):
            # Bootstrap 抽样
            indices = np.random.choice(n_samples, n_samples, replace=True)
            X_boot, y_boot = X[indices], y[indices]

            # 训练一棵树（这里简化——完整的随机森林还需要在每棵树的分裂时做特征随机采样）
            tree = DecisionTree(max_depth=self.max_depth)
            tree.fit(X_boot, y_boot)
            self.trees.append(tree)

        return self

    def predict(self, X):
        # 所有树投票
        predictions = np.array([tree.predict(X) for tree in self.trees])
        # 多数表决
        from scipy import stats
        return stats.mode(predictions, axis=0)[0].flatten()
```

::: tip 场景
「一棵树就像只有一个目击者作证。」我盯着验证集的数字——训练集 98.7%，验证集 91.2%。「但如果我有 100 个目击者——每个人只看到了一小部分证据——他们集体投票的结果，可能比一个专家更稳。」

红莉栖放下咖啡杯。她看我的眼神有点不一样了。

「你刚才独立推出了 Bagging 的基本直觉。Bootstrap Aggregating。Leo Breiman。1996。」

「本天才——」

「别。」

### 2.3.3 Boosting：从错误中学习

Bagging 并行训练——每棵树独立，最后投票。Boosting 则**串行**——每一棵树都看着前面树的错误，专门去修正。

**AdaBoost（1995）**：给错分样本更高的权重。下一棵树的训练目标从"对所有样本一视同仁"变成"更关注之前搞错的样本"。

**Gradient Boosting（1999）**：不再调整样本权重，而是让每棵新树去拟合前一轮的**残差**——"真实值减去前面所有树的预测值之和，还剩多少没拟合好？"

```
第 1 棵树：ŷ₁ = f₁(x)
残差：r₁ = y - ŷ₁

第 2 棵树：拟合 r₁ → ŷ₂ = f₁(x) + ε · f₂(x)
残差：r₂ = y - ŷ₂

第 3 棵树：拟合 r₂ → ŷ₃ = ŷ₂ + ε · f₃(x)
...

最终预测：ŷ = f₁(x) + ε·f₂(x) + ε·f₃(x) + ... + ε·fₘ(x)
```

每一棵树都是弱学习器（通常只分裂 3-6 层），但串行叠加后形成强学习器。学习率 $\varepsilon$ 控制每棵树的贡献——$\varepsilon$ 越小，需要更多棵树，但泛化通常更好。

**工业界三巨头：**

| | XGBoost | LightGBM | CatBoost |
|---|---|---|---|
| **作者** | 陈天奇 (2014) | 微软 (2017) | Yandex (2017) |
| **树生长策略** | Level-wise（按层） | Leaf-wise（挑增益最大的叶子先长） | Symmetric（对称生长） |
| **核心创新** | 正则化目标函数 | GOSS(梯度单边采样) + EFB | Ordered Boosting（对抗预测偏移） |
| **类别特征** | 需手动编码 | 需手动编码 | 原生支持 |
| **速度** | 快 | 更快 | 较慢但更稳 |

::: tip 冷知识
XGBoost 是华盛顿大学博士生陈天奇 2014 年的个人项目。他不是教授，不是大厂研究员，就是一个读博期间写了开源库的人。到 2016 年，Kaggle 上 29 个获奖方案有 17 个用了它。一个人的 side project，两年内统治了整个表格数据竞赛界。2024 年的今天，XGBoost 仍是 Kaggle 表格类比赛的默认 basline。

桶子每次讲这个故事都要加一句：「所以说喵，别等别人给你许可。写就完了。」
:::

## 2.4 为什么这些"老方法"还没死？

在深度学习席卷一切的年代，一个自然的疑问是：学这些干嘛？

### 2.4.1 表格数据的反击

2023 年 NeurIPS 一场 workshop。组织者在 45 个表格数据集上对比了深度学习方法（TabNet、FT-Transformer、SAINT 等）和 XGBoost。

**XGBoost 在 38 个数据集上赢了。**

为什么？表格数据和图像/文本有根本的不同：

| | 图像 / 文本 | 表格数据 |
|---|---|---|
| **特征关系** | 像素之间有空间局部性 | "年龄"和"收入"是两个独立概念 |
| **特征语义** | 单个像素没有语义 | 每一列都有明确的语义 |
| **平滑假设** | 相邻像素通常相似 | "年龄 35"和"年龄 36"的提升可能是完全不同的规律 |
| **擅长的模型** | 深度学习（卷积、自注意力擅长平滑插值） | XGBoost（树模型擅长找异质特征的精确分裂点） |

深度学习的核心操作——卷积和自注意力——依赖"相邻特征之间平滑关联"的假设。当你的特征列之间没有这种平滑关联时，决策树的逐特征精确分裂反而更有效。

### 2.4.2 经典与深度的互补

这不是"谁取代谁"的问题。这是选择对的工具。

| 场景 | 推荐方法 | 原因 |
|------|----------|------|
| 表格数据（~100 列以内） | XGBoost / LightGBM | 异质特征，树模型天然优势 |
| 图像分类 | CNN / ViT | 空间局部性，卷积天然优势 |
| 文本理解 | Transformer | 序列依赖，注意力天然优势 |
| 小数据（<1000 样本） | 经典方法 + 交叉验证 | 深度学习数据不够 |
| 需要可解释性 | 决策树 / 线性模型 | 透明推理链 |

### 2.4.3 伏笔：分而治之的幽灵

而且，这些"老方法"的思想并没有消失——它们换了名字，藏在了最前沿的架构里。

**MoE（Mixture of Experts）**——GPT-4 里可能就在用。核心思想：一个路由器把每个输入分给不同的"专家"网络。

这和决策树做的是同一件事。决策树问"像素 156 > 128 吗？"，把数据分成两路。MoE 的路由器问"这个 token 应该交给专家 A 还是专家 B？"，把计算分成多路。

**分而治之。** 1963 年 Morgan 在打孔卡片上实现的思想，50 年后成了大模型的核心架构。这是 Ch15 分支链的主题。

::: tip 场景
「所以这些方法不是被淘汰了。」我看着自己从 KNN 一路写到随机森林的代码。「它们只是找到了自己最擅长的问题。」

「而且它们的幽灵还在前沿架构里漂着。」红莉栖说。「你知道 MoE 吗——Mixture of Experts？大模型的热门架构。它的路由器本质上就是一棵决策树。把输入分给不同的专家。分而治之。1963 年到 2024 年，同一个思想换了个名字。」

我沉默了一会。

然后开始敲键盘。不是写规则——是写下一章的代码。

> *"El Psy Congroo。"*
:::

## 2.5 章节总结

### 核心概念回顾

| 概念 | 说明 |
|------|------|
| **KNN** | 懒惰学习器——零训练，预测时比距离、投票 |
| **维度灾难** | 高维空间中所有点距离趋同，KNN 失效 |
| **决策树** | 递归二分——每一步找一个特征和阈值最大化纯度提升 |
| **信息增益** | 分裂前后的熵减少——用 Shannon 1948 的公式选最佳分裂 |
| **基尼不纯度** | 分裂标准的替代方案——比熵计算更快 |
| **剪枝** | 预剪枝（限制深度）/ 后剪枝（先长满再砍）——防止过拟合 |
| **Bagging** | Bootstrap 抽样 → 并行训练 → 投票降低方差 |
| **随机森林** | Bagging + 特征随机采样 → 树之间尽量不一样 |
| **Boosting** | 串行训练——每棵树专门修正前树的错误 |
| **残差拟合** | Gradient Boosting 的核心——每棵新树拟合的是"还剩多少没学好" |
| **XGBoost** | 工业界表格数据王者——正则化 + 二阶近似 + 并行优化 |

### 架构演进

```
KNN (1951)
 │   懒惰学习，零训练
 │   维度灾难 + 预测慢
 │
 ▼
决策树 (1963)
 │   分而治之，可解释
 │   单棵树容易过拟合
 │
 ▼
Bagging (1996) → 随机森林 (2001)
 │   Bootstrap + 投票 → + 特征随机
 │   降低方差
 │
 ▼
Boosting (1995→1999→2014)
     AdaBoost → Gradient Boosting → XGBoost
     调整权重 → 拟合残差 → 工业级实现
```

### 章节要点

1. **KNN 是学习的零形式**——什么都不做，只在需要时查询。维度灾难是它无法跨越的天花板。
2. **决策树把分类变成了问答游戏**——每一步选一个能把数据最干净切开的"是否"问题。可解释性是它最大的资产。
3. **集成方法把弱变强**——Bagging 靠随机性（方差降低），Boosting 靠修正错误（偏差降低）。
4. **在表格数据上，经典方法还没有输**——38/45。不是怀旧，是事实。
5. **分而治之没有死**——MoE 是决策树在大模型时代的还魂。
