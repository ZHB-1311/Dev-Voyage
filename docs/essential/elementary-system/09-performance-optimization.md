# 第 9 章：电脑卡了？先别砸！

> 大一开学，你买了台游戏本。开机 10 秒，LOL 200 帧，风扇安静得像猫。三个月后，开机 1 分钟，游戏掉到 60 帧，风扇起飞。再过半年，开个 Word 都要等 10 秒，C 盘红了，桌面堆满文件，右下角一排弹窗。
>
> 你试过三个"电脑管家"、两次重装、一个散热底座。没有用。
>
> 问题不在"电脑老了"——你的硬件性能跟买来那天一样。问题在别处。

## 安迪-比尔定律：你的性能被谁偷走了

计算机界有一个著名的定律叫"安迪-比尔定律"（Andy and Bill's Law）。

安迪是指英特尔的前 CEO 安迪·格鲁夫，比尔是指微软的比尔·盖茨。定律的内容是：**"安迪提供什么，比尔就拿走什么。"**（What Andy gives, Bill takes away.）

意思是：英特尔辛辛苦苦把硬件性能提升了一倍，微软（以及所有软件厂商）就会开发出更庞大、更吃资源的软件，把这些新增的性能全部吃掉。

这就是为什么你的电脑明明比十年前快了 100 倍，打开 Word 的速度感觉还是差不多。

::: tip 一个数字
1995 年，比尔·盖茨曾说："640KB 内存对任何人都够了。"这句话后来被反复引用——多半是断章取义，但数字本身足够震撼。

2024 年，Chrome 浏览器开 10 个标签就能用掉 4GB 内存。640KB × 6400 倍。软件开发者知道内存便宜了，写代码时就不再精打细算——反正用户会升级硬件。
:::

::: info 安迪-比尔定律的推论
"安迪-比尔定律"解释了消费电子行业最底层的驱动力：**你的旧设备不会变慢，但新软件会越来越重。** 硬件厂商和软件厂商之间有一种微妙的共生关系——英特尔需要微软把新 CPU 的性能吃光，这样你才会买下一代的 CPU；微软需要英特尔的 CPU 越来越快，这样它的新功能才跑得动。

这个循环对行业是增长引擎，对消费者是"计划性淘汰"。你手里的设备没有变差，是生态逼着它变差。
:::

## 电脑为什么会变卡

### 硬件老化？不完全是

一个常见误解是"电脑用久了，硬件会老化变慢"。

真相：CPU、内存、硬盘这些核心硬件，性能不会随时间下降。CPU 的晶体管不会"疲劳"，内存的电容不会"生锈"——要么正常工作，要么直接坏掉，不存在"变慢"这个中间状态。

那为什么旧电脑就是感觉慢？软件越来越重，硬件相对就弱了。就像 10 年前的车还能开，但今天的道路限速提高了，你跑不动了。

::: info 一个例外：硅退化
CPU 在高电压高温下长期工作，确实存在一个叫"硅退化"的现象——晶体管的开关特性会轻微漂移。但这个效应极其微小，一块正常使用的 CPU 在 10 年内性能下降通常不到 1%，远比软件膨胀的影响小。所以"CPU 老了变慢"在工程上基本是伪命题。
:::

### 卡顿的四大元凶

::: warning 第一：存储空间不足
**症状**：开机慢，打开软件慢，保存文件慢。

**原理**：硬盘需要留出 15-20% 的空白空间作为缓冲。机械硬盘需要连续空间来减少碎片，SSD 需要空白块来做写入均衡和垃圾回收。

**临界点**：C 盘剩余空间 < 总容量 10%——到了这个点，性能断崖式下跌。
:::

::: warning 第二：后台程序太多
**症状**：什么都没干，风扇狂转，内存占用 80%+。

**元凶**：各种软件的"开机自启动"、"后台服务"、"自动更新"。典型代表：Adobe Creative Cloud、Google Update、腾讯系防护进程、各种 OEM 预装的管理软件。
:::

::: warning 第三：软件膨胀
**症状**：同样的软件，新版比旧版占用更多资源。

例子：
- Chrome 浏览器：2010 年内存占用 200MB，现在开 5 个标签就 1GB
- 微信：从聊天工具演变成了"操作系统"
- Office：功能越来越多，启动越来越慢
- 显卡驱动：NVIDIA 驱动安装包从 2016 年的 300MB 膨胀到现在的 800MB+
:::

::: warning 第四：散热受限
**症状**：用一会儿就卡，特别是夏天或玩游戏时。

**原理**：CPU 和 GPU 有温度墙——超过一定温度自动降频保护。CPU 可能从 4.5GHz 降到 2.0GHz，GPU 从 1800MHz 降到 800MHz。降频后的性能损失可达 40-60%。

**常见场景**：笔记本放在床上堵了进风口、几年没清灰、硅脂干固。
:::

## 存储空间：C 盘红了怎么办

### 各系统查看存储空间

**Windows：**
1. 按 `Win + E` 打开文件资源管理器
2. 在"此电脑"里看 C 盘剩余空间
3. **详细分析**：设置 → 系统 → 存储 → 点击 C 盘

**macOS：**
1. 点击左上角苹果标志 → 关于本机 → 存储
2. **详细分析**：关于本机 → 存储 → 管理

**Linux：**
```bash
df -h  # 查看所有磁盘空间
du -sh *  # 查看当前文件夹大小
```

**手机（安卓/iOS）：**
- 设置 → 通用 → iPhone 存储空间 / 设置 → 存储

### WizTree：1 秒扫描整个硬盘

Windows 自带的"存储"页面能告诉你哪个文件夹大，但速度慢，粒度粗。真正好用的工具是 WizTree。

**WizTree 为什么快？**

传统磁盘分析工具逐个文件夹扫描，1TB 硬盘要等好几分钟。WizTree 直接读 NTFS 文件系统的 MFT（Master File Table，主文件表）——这是 Windows 自己维护的文件索引，记录了每个文件的位置和大小。读 MFT 只需要几秒钟，大小跟硬盘容量无关。

打开 WizTree，你会看到一个**树状方块图**：每个方块代表一个文件，面积正比于文件大小。一眼就能看出谁是"空间大户"——不需要任何技术知识。

**免费下载**：diskanalyzer.com

### 手动识别：这个文件夹能删吗？

WizTree 告诉你了"什么东西大"，但没有告诉你"能不能删"。以下是确定性最高的清理目标：

**确定能删的：**
- `C:\Windows\Temp` — 系统临时文件，全删
- `%TEMP%`（Win+R 输入这个）— 用户临时文件，全删
- `C:\Users\你的名字\AppData\Local\Temp` — 同上
- 回收站 — 清空

**能删但要注意的：**
- **微信/QQ 聊天文件缓存**：`C:\Users\你的名字\Documents\WeChat Files\你的微信号\FileStorage` — 这里是所有聊天记录里的文件、图片、视频。如果云上有备份，可以清理。有些人这里占了几十 GB。
- **QQ 接收的文件**：`C:\Users\你的名字\Documents\Tencent Files\你的QQ号\FileRecv` — 同理
- **npm/yarn 缓存**（如果你写前端）：`C:\Users\你的名字\AppData\Local\npm-cache` — 可以删，下次安装时会重新下载
- **pip 缓存**（如果你写 Python）：`C:\Users\你的名字\AppData\Local\pip\cache` — 同理
- **Steam 游戏安装目录**：`C:\Program Files (x86)\Steam\steamapps\common` — 在 Steam 里右键卸载不玩的游戏，比手动删文件夹安全

**绝对不能删的：**
- `C:\Windows` — 系统核心文件
- `C:\Program Files` 和 `C:\Program Files (x86)` 里的任何你认不出来的东西 — 用正规卸载，不要手动删
- `C:\Users\你的名字\AppData` 里的大部分内容 — 除非你确定是临时文件或缓存

### 用 AI 辅助判断（可选）

面对一个不认识的大文件夹，最安全的方式是先问问。你可以这样做：

1. WizTree 里看到 `C:\ProgramData\Package Cache` 占了几十 GB，不知道是啥
2. 截图或粘贴路径到 ChatGPT / Claude / Kimi，问"这个文件夹是干什么的？能删吗？"
3. AI 会告诉你：这是 Visual Studio 的安装包缓存，删了之后 VS 的修复功能会受影响，但如果你不打算修复 VS，可以安全删除

**AI 在这里的作用**是加速识别——它帮你省掉搜索引擎翻三页的时间。但它不能替你决定，最终判断还是你自己的。

::: caution AI 给出的建议要交叉验证
AI 对某些冷门文件夹的判断可能出错。涉及删除操作时，一个额外保险的做法是：把文件夹改名（加个 `.bak` 后缀），等一两周确认没出问题再彻底删除。
:::

### 图吧工具箱：一站式诊断套装

如果你不想一个一个地找工具——WizTree 看磁盘、HWMonitor 看温度、CPU-Z 看硬件、CrystalDiskInfo 看硬盘健康、MSI Afterburner 调显卡——有一个更方便的选择：图吧工具箱。

图吧工具箱源于"图拉丁吧"（国内 PC 硬件爱好者社区），把几十款免费硬件工具打包在一起，一键即可使用。CPU 检测、内存测试、硬盘扫描、显卡烤机、屏幕坏点检测，都在里面。

大学生开箱新笔记本验机、二手硬件检测、日常故障排查——一个工具包就够了。

**获取方式**：官网直接下载（免费），注意避开各种下载站的"高速下载"陷阱——这个你在[第 3 章](./03-installing-software.md)已经学会了。

### 快速清理：这些可以删

**Windows 用户必删清单：**
1. **临时文件**：`C:\Windows\Temp` 和 `%TEMP%`（按 Win+R 输入 `%TEMP%`）
2. **下载文件夹**：`C:\Users\你的名字\Downloads`
3. **旧系统备份**：设置 → 系统 → 存储 → 临时文件 → 以前的 Windows 安装
4. **休眠文件**（如果你不用休眠）：以管理员运行 CMD：`powercfg -h off`
5. **回收站**：右键回收站 → 清空回收站

**macOS 用户必删清单：**
1. **下载文件夹**：`~/Downloads`
2. **废纸篓**：右键 Dock 栏废纸篓 → 清空
3. **iOS 备份**：`~/Library/Application Support/MobileSync/Backup`
4. **Xcode 缓存**（如果不用 Xcode）：`~/Library/Developer/Xcode/DerivedData`

**通用清理工具：**
- **Windows**：自带的"磁盘清理"工具（搜索"磁盘清理"）
- **macOS**：自带的"储存空间管理"
- **第三方**：BleachBit（开源免费）、CCleaner（有免费版但小心捆绑）

### 高级技巧：移动这些文件夹

**Windows 移动"文档"、"下载"、"桌面"：**
1. 右键"文档"文件夹 → 属性 → 位置
2. 点击"移动"，选择 D 盘或其他分区的新位置
3. 系统会问是否移动现有文件，选"是"

**原理**：这些文件夹默认在 C 盘，把它们移到其他盘，给 C 盘腾空间。

**macOS 类似操作：**
1. 把"下载"、"文档"文件夹拖到外置硬盘
2. 按住 Cmd+Option 拖，会创建替身（快捷方式）

## 后台程序：谁在偷偷吃资源

### 找出"资源吸血鬼"

**Windows 任务管理器（按 Ctrl+Shift+Esc）：**
1. 看"进程"选项卡，按"内存"或"CPU"排序
2. **重点检查**：内存占用 > 200MB，CPU 持续 > 10% 的程序
3. **可疑程序**：名字奇怪的各种"助手"、"更新服务"

**macOS 活动监视器（应用程序 → 实用工具）：**
1. 看"内存"和"CPU"选项卡
2. **重点检查**：%CPU 持续高的进程

**Linux：**
```bash
top  # 实时查看进程
htop  # 更好用的 top（需要安装）
```

### Windows 服务：那个 svchost.exe 是啥

打开任务管理器，你会看到一堆叫 `svchost.exe` 的进程，每个占用几十到几百 MB 内存。每个 `svchost.exe` 都是 Windows 服务的宿主——把自己的功能包装成"随系统启动、不需要用户登录就在后台跑"的东西。

**服务和普通程序的区别：**
- 普通程序：你双击启动它，关掉就停了
- 服务：开机自动启动，不需要用户登录，一直跑着，你关不掉它
- 常见服务：Windows Update、打印服务、网络发现、蓝牙

**什么时候服务出问题？** 某个 svchost.exe 持续占用大量 CPU——这时候要看这个 svchost 下面挂了什么服务。任务管理器里右键 svchost.exe → 转到服务，就能看到它托管的所有服务，然后查哪个在搞鬼。

**msconfig 管理服务：**
1. 按 `Win + R`，输入 `msconfig`
2. 服务选项卡 → 勾选"隐藏所有 Microsoft 服务"
3. 剩下的可以酌情禁用——但要确保你知道自己在关什么

### Sysinternals：微软给你的手术刀

微软有一整套免费的系统诊断工具叫 Sysinternals，由 Mark Russinovich（现在已经是微软 Azure CTO）开发。其中最实用的两个：

**Autoruns：**
任务管理器的"启动"选项卡只显示了一小部分自启动入口。Autoruns 能看到十几类启动入口——登录项、Explorer 插件、IE 插件（是的，这些 20 年前的插件钩子还在影响系统）、计划任务、服务、驱动、网络提供程序、Winsock 提供程序。很多流氓软件和广告程序不在"启动"里，在"计划任务"和"服务"里——Autoruns 都能看到。

**Process Explorer：**
任务管理器的终极版。能显示进程的父子关系（谁启动了谁），能看每个进程打开了哪些文件、加载了哪些 DLL、占用了哪些句柄。Windows 常见的一个痛点：你想删某个文件，系统说"文件被占用，无法删除"——Process Explorer 可以搜到是哪个进程在占用这个文件。

**怎么获取**：直接搜 "Sysinternals" 进微软官网下载，或者用 `winget install Microsoft.Sysinternals`。

### 禁用开机自启动

**Windows 10/11：**
1. 任务管理器 → 启动选项卡
2. 右键不需要的程序 → 禁用
3. **保守原则**：只禁用你认识的非必要程序

**macOS：**
1. 系统设置 → 通用 → 登录项
2. 删除不需要开机启动的程序

**Linux（Ubuntu）：**
```bash
sudo systemctl list-unit-files --type=service  # 查看所有服务
sudo systemctl disable 服务名  # 禁用服务
```

### 浏览器：内存杀手第一名

**Chrome 内存查看：**
1. 按 `Shift + Esc` 打开 Chrome 任务管理器
2. 你会惊讶：一个标签页可能占用 500MB 内存

**浏览器省内存技巧：**
1. **扩展程序管理**：`chrome://extensions/`，关掉不用的
2. **标签页管理**：用 OneTab 等扩展把不用的标签页"冻结"
3. **硬件加速**：设置 → 系统 → 开启硬件加速（用 GPU 减轻 CPU 负担）

## 软件膨胀：如何选择轻量软件

### 常见软件的"轻量替代品"

| 需求 | 主流软件（重） | 轻量替代品（轻） |
|:---|:---|:---|
| **办公套件** | Microsoft Office（2GB+） | WPS Office（200MB）、LibreOffice |
| **图像处理** | Photoshop（3GB+） | GIMP（300MB）、Paint.NET |
| **视频播放** | 暴风影音（带广告） | VLC（100MB）、MPC-HC |
| **压缩软件** | WinRAR（带广告） | 7-Zip（2MB）、Bandizip |
| **浏览器** | Chrome（内存杀手） | Firefox、Edge（现在其实不错） |

### 如何判断软件是否"太重"

**安装包大小**：
- 轻量：< 50MB
- 中等：50-200MB
- 重量级：> 200MB

**内存占用**：
- 轻量：< 100MB
- 中等：100-500MB
- 重量级：> 500MB

**启动时间**：
- 轻量：< 3 秒
- 中等：3-10 秒
- 重量级：> 10 秒

### 绿色软件/便携软件

**概念**：不需要安装，解压就能用，不写注册表，不污染系统。

**推荐站点**：
- PortableApps.com：各种便携软件
- 小众软件：国内推荐绿色软件的博客

**优点**：不污染系统，可以放在 U 盘里随身带，卸载直接删文件夹就行。

**缺点**：可能没有自动更新，有些功能可能缺失。

## 游戏专项优化

大学宿舍里，游戏本的普及率极高。一台一万块的拯救者，出厂状态打了八折——很多性能被默认设置白费了。

### 画质选项：什么值得开，什么应该关

游戏里那几十个画质选项，不是每一项都值得拉满。很多选项从"超高"降到"高"，肉眼根本看不出区别，但帧率能涨 30-50%。

**高损耗、低感知——建议优先降到中/高：**

| 选项 | 性能损耗 | 视觉影响 | 建议 |
|:---|:---|:---|:---|
| 阴影质量 | 高（15-25%） | 低了只是影子边缘模糊一点 | 中或高，别开超高 |
| 体积云/体积雾 | 极高（20-30%） | 大多数时候你根本不会看云 | 中或低 |
| 屏幕空间反射 | 高（10-20%） | 水面和光滑表面的倒影精度 | 中 |
| 环境光遮蔽 | 中（5-15%） | 物体交界处的柔和阴影 | 高（性价比较好） |
| 景深 | 中（5-10%） | 远处模糊效果——竞技玩家直接关 | 关或低 |
| 动态模糊 | 低（3-5%） | 转视角时的模糊拖影——很多人主动关 | 关 |
| 光线追踪 | 极高（30-50%） | 笔记本上开了基本告别流畅 | 关 |

**低损耗、高感知——建议拉满：**

| 选项 | 性能损耗 | 视觉影响 | 建议 |
|:---|:---|:---|:---|
| 纹理质量 | 几乎为零（只要有足够显存） | 高低差距一眼可见 | 高或超高 |
| 各向异性过滤 | < 1% | 远处地面纹理清晰度 | 16x |
| 视野距离 | 低（主要吃 CPU） | 远处物体是否显示 | 高 |
| 抗锯齿（2x/4x） | 中（5-10%） | 物体边缘是否平滑 | 4x 足够，8x 浪费 |

**竞技类游戏（LOL、Valorant、CS2）特别推荐：** 关掉垂直同步（它会限制帧率并增加输入延迟）、关掉动态模糊和景深。这几种效果在竞技场景里唯一的作用是降低你的 KDA。

### 游戏本的硬件设计：比你想象的讲究

游戏本不只是"把台式机零件塞进笔记本"。有几个关键设计决定了你的游戏体验，但很多人买了两年都不知道。

**性能模式切换：**

所有游戏本都有多种性能模式——但各品牌叫法不同。联想叫"安静/均衡/野兽"，ROG 叫"静音/性能/增强"，惠普叫"省电/均衡/狂暴"。

本质是一样的：
- **安静/静音模式**：风扇低速，CPU 限制在 15-25W，图书馆里用不吵人
- **均衡模式**：风扇中速，CPU 约 30-45W，日常使用够
- **狂暴/野兽模式**：风扇满速，CPU 能拉到 60-80W+，GPU 也解禁功耗上限

**狂暴模式必须插电**——这是物理限制。电池供电时，CPU+GPU 加起来最多 50-70W（保护电池）；插上电源，能跑到 150-200W。这不是厂商的营销套路，是锂电池的放电能力有限。

很多人的游戏本一直用均衡模式——没切过模式，等于买了台 8000 块的电脑只用了 5000 块的性能。

**MUX 开关：独显直连**

游戏本有两块显卡：CPU 自带的核显（省电）和 NVIDIA/AMD 独立显卡（高性能但费电）。默认情况下，独显渲染的画面要经过核显才输出到屏幕——这个"中转"叫 Optimus 或混合模式。

问题：这个中转环节会吃掉 10-20% 的帧率，而且是永久损耗——不管你玩什么游戏。

解决方案：**独显直连**。在品牌控制中心（联想电脑管家、ROG Armoury Crate、惠普 OMEN Gaming Hub）或 NVIDIA 控制面板里切换。近几年主流游戏本都支持，但出厂默认通常是混合模式。切到独显直连，帧率白涨 10-20%。

**双显三模：**

现在大多数游戏本支持三种显示模式：
1. **独显直连**：最高性能，最费电。插电打游戏用。
2. **混合模式**：核显负责日常，独显按需启动。一般用这个。
3. **纯集显模式**：独显完全关闭。图书馆续航从 2 小时涨到 5-6 小时。

三种模式在品牌控制中心里切换，通常需要重启。如果 MUX 开关是硬件级的（Advanced Optimus），切换时不用重启——但这个功能只在高端型号上有。

### 降频的真相：撞墙

当你游戏打着打着帧率突然暴降——你撞墙了。不是网络，是功耗墙或温度墙。

- **温度墙**：CPU/GPU 温度超过某个阈值（通常是 CPU 95°C，GPU 87°C），自动降频。频率砍一半，帧率砍更多。
- **功耗墙**：CPU 长期功耗被限制在某个值（比如 PL1 45W）。即使温度只有 70°C，CPU 的功耗也不会超过这个数——不是散热不够，是厂商设的线。

温度墙靠清灰、换硅脂、加散热底座解决。功耗墙靠切性能模式（狂暴模式会抬高这个限制）或者更进一步——降压。

::: info 为什么游戏本风扇响得像直升机
游戏本的风扇转速可以到 5000-7000 RPM。一个对比：台式机风扇通常 1000-2000 RPM，电风扇最高档约 1200 RPM。

游戏本用如此高的转速，是因为**物理定律不允许它既薄又安静**。机身空间只够放薄型涡轮风扇和细热管，要把 150W 的热量排出去，唯一的选择是疯狂吹。这是工程上的取舍，不是厂商偷懒。
:::

## CPU 和 GPU 优化：往上限靠拢

这部分是进阶内容。不是必须学的——但如果你的游戏本打过降频后性能还是差，这些技巧能让你榨出最后 10-20% 的性能。

### CPU 降压（Undervolting）

很多游戏本的 CPU 默认电压偏高——厂商为了保证每一颗 CPU（包括体质最差的那颗）都能稳定运行，给的是保守值。

降压就是降低 CPU 核心电压，但不降低频率。电压每降低 50mV，功耗能降 5-10W。功耗降了，温度就降，温度降了，CPU 就能维持更高频率更久——也就是说，降压反向提升了性能。

**工具：**
- **Intel CPU**：ThrottleStop 或 Intel XTU
- **AMD CPU**：Ryzen Master 或在 BIOS 里调 Curve Optimizer

**操作逻辑：**
1. 从 -50mV 开始，跑一轮压力测试（Cinebench 或者你平时玩的游戏）
2. 稳定就继续降 -10mV，直到蓝屏或不稳定
3. 回退 10-15mV，用这个值

**安全结论：** 降压不会损坏硬件。电压低了，温度低了，反而延长寿命。最坏的结果是不稳定蓝屏——调回去就好了。

### 体质差异：硅彩票

两个同学买了一模一样的拯救者 Y9000P，出厂日期差三个月。A 同学的 CPU 能降压 150mV 稳定跑，B 同学的降 60mV 就开始蓝屏。

这就是"硅彩票"（Silicon Lottery）——同一型号的 CPU，每颗的体质都不一样。晶圆制造过程中的微小差异导致了漏电率的分布。所有 CPU 都达到了出厂标称性能，但降压余量就看运气了。

这是硬件世界的真实面，不存在"同型号就一样"。

### GPU 降压超频

GPU 跟 CPU 一样，默认电压偏保守。但显卡有一个额外的特征：NVIDIA GPU 有 GPU Boost 机制，温度越低，自动 Boost 频率越高。

所以 GPU 的降压逻辑更优雅：**降低核心电压 + 拉高频率曲线 = 降压超频**。核心思路是"少给电压、多给频率"。

**工具**：MSI Afterburner（免费，所有品牌显卡都能用）。

**操作**：按 `Ctrl+F` 打开电压/频率曲线编辑器，把目标频率对应的电压往下拉，然后拉平过高的频率段。这个操作不复杂，B站搜"30 系降压"或"40 系降压"有详细教程。

**笔记本 GPU 的额外收益**：笔记本显卡功耗限制更严（Max-Q 设计的延续），每次降压省出的那几瓦，恰好能让频率多在 Boost 区间待一会儿。所以笔记本显卡的降压收益通常比台式机更明显。

::: caution 降压不是超频
超频是提高频率，可能增加电压和热量，有损坏风险。降压是降低电压、保持或优化频率——不增加功耗，不增加热量，不降低寿命。这是"免费午餐"。

但如果你在 BIOS 里乱改电压偏移方向（设成了正向偏移），那确实会损坏 CPU。操作前看清楚选项名称。
:::

## 内存和存储：怎么优化

### 内存：不只看"已用"，还要看"已缓存"

打开任务管理器，看到内存"已用 80%"，你以为快不够了。但 Windows 10/11 的内存管理比以前聪明：

- **正在使用**：程序实打实在用的内存
- **已修改**：还没写入磁盘的数据
- **备用（Standby）**：已经被压缩暂存，但随时可以恢复给程序用
- **空闲**：真的没用

备用内存里有很多是 Windows 的内存压缩——系统把不活跃的数据压缩存着（压缩比通常 2:1 到 3:1），需要的时候瞬间解压恢复。所以你看到的"占用高"中相当一部分是压缩缓存，需要时可以被释放。

**结论**：Windows 的内存管理在现代版本（10/11）已经足够好。不要装"内存清理大师"之类的软件——它们做的事情是清空备用缓存，实际上让系统变慢，因为这些缓存被清掉后下次还要重新从磁盘读。

### SysMain（Superfetch）：不该关

Windows 的 SysMain 服务（以前叫 Superfetch）会学习你的使用习惯，在开机时预加载你常用的程序到内存备用缓存。很多人看到"内存占用高"就搜到教程关了它——

关掉 SysMain 的后果：打开常用程序变慢，而内存确实少占了几百 MB——这几百 MB 对你 16GB 的电脑来说毫无意义。

**一个总原则**：不要为了"好看"而优化。任务管理器里的数字好看，不代表你用得流畅。

### ReadyBoost：一段被遗忘的历史

Windows Vista 时代，微软推出了一个功能叫 ReadyBoost：插一个 U 盘，系统把它当缓存用。当时的背景是——内存巨贵，很多人只有 512MB；U 盘的随机读取比机械硬盘快。

结果：U 盘的速度远不如实际内存，而且频繁写入会把 U 盘写废。SSD 普及后，这个功能已经完全没用了。如果在某个教程里看到"用 U 盘加速电脑"——那篇教程比你年轻。

### 虚拟内存：让系统自己管

虚拟内存（页面文件）是当物理内存不够时，把一部分数据暂存到硬盘上。它慢——SSD 的随机读写速度大约是 DDR4 内存的 1/100。但它是安全网。

**现代的建议是让系统自动管理。** 如果物理内存足够（16GB+），页面文件实际很少被用到。但关掉它会让某些老程序崩溃——因为它们假定页面文件一定存在。

曾经流传的"虚拟内存 = 物理内存 × 1.5 或 × 3"公式来自 1990 年代——当时内存以 MB 计，Windows 的页面管理算法也原始。现在的内存管理策略完全不同，手动设反而可能设错。

### 存储感知（Storage Sense）

Windows 10/11 内置了自动化清理：设置 → 系统 → 存储 → 存储感知。开启后可以设定"当剩余空间少于 X GB 时自动清理临时文件"。

一个实用设置：把下载文件夹的自动清理设为"30 天未更改"，回收站自动清理设为"30 天"。这两项一个月触发一次，基本不会干扰正常使用。

### Compact OS：把系统压缩一下

Windows 自带一个叫 Compact OS 的工具，能把系统文件压缩存储。压缩比通常 1.5:1 到 2:1——一个 20GB 的系统目录能压到 10-12GB。解压是透明的，运行时 CPU 自动解压，性能影响微乎其微（现在 CPU 解压速度远大于磁盘读取速度）。

以管理员身份运行 CMD：
```cmd
compact /compactos:always
```

能省几个 GB 到十几 GB，对 128GB 或 256GB 小容量笔记本特别实用。

### NTFS 压缩：不推荐，但知道它在干什么

右键文件夹 → 属性 → 高级 → "压缩内容以节省磁盘空间"——这是 NTFS 文件级的透明压缩。跟 Compact OS 原理一样，但粒度是文件和文件夹。对文本和日志文件效果好，对图片和视频几乎没用（这些文件本身已经是压缩格式）。

**不推荐的原因**：容易忘记哪些文件夹开了压缩，以后拷到别处可能困惑。而且 Windows 在资源管理器里默认用蓝色文字标记压缩过的文件，看起来像中毒了。除非你的 C 盘真的只剩几 GB 且没有别的办法。

## 散热问题：给电脑退烧

### 判断是否过热

**Windows 查看温度：**
- HWMonitor（免费软件）
- Core Temp（专门看 CPU 温度）
- 图吧工具箱里带了这些

**macOS：**
- iStat Menus（付费但强大）
- Fanny（免费，在通知中心显示温度）

**临界温度：**
- CPU：> 85°C 开始降频，> 95°C 危险
- GPU：> 80°C 开始降频，> 90°C 危险

### 物理清灰

**笔记本清灰步骤：**
1. 准备工具：螺丝刀、刷子、吹风机（冷风档）
2. **断电**：拔电源，拆电池（如果可拆）
3. 拆后盖：找到散热风扇位置
4. 清灰：用刷子轻轻刷，吹风机吹走灰尘
5. 装回去：螺丝别弄丢，别拧太紧

**如果不敢拆：**
1. 用吹风机冷风档吹散热口
2. 买个散热底座（有点用，但有限——降 2-5°C 顶天了）
3. 找专业人士清灰（50-100 元）

::: info 硅脂：容易忽略的耗材
CPU 和散热器之间的导热介质叫硅脂。它的特性是：时间久了会干固，导热效率大幅下降。

新电脑的硅脂能用 2-3 年。如果你发现清灰后温度还是高，风扇满速但出风口的风不热——说明硅脂干了，热量传不到散热片上。这时候需要换硅脂。

换硅脂不难，但需要拆散热器。如果你没把握，掏点钱找电脑店。好的硅脂（信越 7921、利民 TFX）几十块钱一支。
:::

### 软件降温

**降低性能换温度：**
- **Windows 电源计划**：设置 → 系统 → 电源和睡眠 → 其他电源设置 → 选择"节能"
- **游戏本控制中心**：大多数游戏本有控制软件，可以调性能模式
- **限帧**：游戏里限制最高帧率（比如从 144 帧限到 60 帧），GPU 负载骤降，温度立竿见影

## 系统修复：重装之前还有救

大多数人遇到系统问题——偶尔蓝屏、开机变慢、某个功能莫名其妙失效——第一反应是重装。但重装是大炮，大多数问题用螺丝刀就够了。

### DISM 和 SFC

Windows 自带两个系统修复命令：

**SFC（System File Checker）：**
```cmd
sfc /scannow
```
扫描所有受保护的系统文件，如果发现损坏或被篡改，自动从本地缓存中恢复正确版本。跑一遍只要 5-15 分钟。

**DISM（Deployment Image Servicing and Management）：**
```cmd
dism /online /cleanup-image /restorehealth
```
如果 SFC 无法修复（因为本地缓存也坏了），DISM 从 Windows Update 服务器拉取健康的文件替换本地损坏的。需要联网，耗时 10-30 分钟。

**使用顺序**：先跑 `sfc /scannow`，如果它说"发现损坏但无法修复"，再跑 DISM 拉健康文件，跑完 DISM 后再跑一次 SFC。

这两个命令能解决一大类"系统莫名其妙出问题"的情况，省下一次重装的时间。

::: tip 什么时候跳过修复直接重装
如果系统已经无法正常启动、或者你中了不太干净的东西杀不干净——直接重装。花两个小时重装比花四个小时排查划算。
:::

## 重装系统：电脑的终极重置

### 什么时候该重装

**重装系统的正确时机：**
1. 系统崩溃，无法正常启动
2. 中了严重病毒，杀不干净
3. 用了 3 年以上，积累了太多垃圾
4. 卖电脑前清空数据

## 从碎片整理到 SSD 革命

### 机械硬盘时代的"碎片整理"

**什么是磁盘碎片？**
早期机械硬盘：文件被分成小块，散落在磁盘各处。读取时磁头需要到处移动，速度变慢。整理就是把文件碎片拼在一起，让磁头少跑路。

**Windows 的经典画面：**
Windows 98/XP 时代：碎片整理进度条能看一小时。人们真的会看着进度条发呆。整理一次能提升 20-30% 速度。

**SSD 时代的变革：**
SSD 没有机械部件，寻道时间恒定为 ~0.1ms，不存在"碎片导致寻道变慢"的问题。碎片整理对 SSD 不仅无用，还会大量写入——减少 SSD 寿命。Windows 10/11 对 SSD 自动禁用碎片整理，改为执行 TRIM 优化。

### SSD 寿命：TBW 和写入放大

SSD 确实有寿命上限，但绝大多数人用不到死。衡量标准是 TBW（Total Bytes Written，总写入量）——这块盘在寿命周期内能写入多少数据。

一块 512GB 的主流 SSD，TBW 通常在 300-600TB。600TB 是什么概念？每天写入 100GB，可以写 16 年。大部分大学生的写入量远低于这个数。

真正伤害 SSD 寿命的元凶：
- 碎片整理软件强制全盘擦写
- BT 下载直接写 SSD（大量随机小写入）
- 把 SSD 写到快满（剩下不到 5% 空间时，写入放大效应暴增）

**CrystalDiskInfo** 能看 SSD 的健康度、已写入量、通电时间。图吧工具箱里的 CrystalDiskInfo 已经配好了中文界面。

### 内存价格的"过山车"

**历史价格：**
- 1990 年：1MB 内存 ≈ 500 美元（今天价值 1000 美元）
- 2000 年：128MB 内存 ≈ 100 美元
- 2010 年：4GB 内存 ≈ 100 美元
- 2020 年：16GB 内存 ≈ 100 美元（疫情期间涨过，后来回落）

这个趋势回答了"为什么老程序员什么都要省着用"——他们当年按 KB 付费，你对 16GB 毫无感觉。两代人对"资源"的本能反应完全不同，根源就在这里。

### 摩尔定律的减速

**摩尔定律（1965 年提出）：**
每 18-24 个月，芯片上的晶体管数量翻一倍。

**曾经的效果：**
- 1995-2005：CPU 速度每年提升 50%
- 2005-2015：提升降到 20%
- 2015-现在：提升降到 10%

**为什么变慢？**
- 物理极限：晶体管不能再小了（现在 3 纳米，再往下靠近原子尺度）
- 散热瓶颈：频率不能再高了（现在 5GHz 是单核上限，再高就烧了）
- 电力限制：手机电池撑不住更快的芯片

**结果**：现在电脑性能提升主要靠：
1. 更多核心（从单核到 8 核、16 核）
2. 专用芯片（GPU、NPU、TPU）
3. 软件优化和多核并行

### 从 Norton Utilities 到不需要"管家"

Windows 95 时代，有一款软件叫 Norton Utilities——最早的"电脑管家"：清理磁盘垃圾、修复注册表错误、加速系统启动。售价 99 美元（相当于今天的 200 美元）。

后来微软在 Windows 98 中内置了类似功能，Norton 就没什么人买了。这个故事的核心：**当操作系统逐渐把常用工具内建，"外挂型电脑管家"的历史使命就结束了。** 今天 Windows 10/11 自带的 Defender、磁盘清理、存储感知、任务管理器——本质上就是当年的 Norton Utilities。

现在你明白为什么那些国产"电脑管家"都在努力做差异化了吧？基础功能微软都内置了，免费且没有弹窗。

## 防卡顿的日常习惯

### 安装软件时的原则

**不安装**：不认识、不需要、不信任的软件
**不勾选**：安装时的附加选项（浏览器主页、捆绑软件）
**不盲目放 C 盘**：保持默认即可——但如果你的 C 盘本身就小（128GB/256GB），安装时手动改到 D 盘

### 每月一次的例行检查

1. 清理临时文件（WizTree 扫一遍 C 盘，看有没有新生的大户）
2. 更新软件（但不立刻追系统大版本——等一个月看反馈）
3. 检查启动项（有没有新装软件偷偷加了自己）
4. 备份重要文件（到移动硬盘或云盘）

### 硬件升级：比任何优化都管用

**升级优先级：**
1. **加内存**（最立竿见影）：8GB → 16GB，多任务流畅度翻倍。但超过 32GB 对 99% 的人没意义
2. **换 SSD**（第二有效）：如果你的电脑还在用机械硬盘做系统盘，换成 SSD 是从自行车换到电瓶车。开机从 1 分钟变 10 秒
3. **换 CPU/显卡**（最贵且受限）：笔记本通常换不了，台式机需考虑主板兼容性。往往不如整机换新

**如何判断该升级什么：**
- 任务管理器里：内存经常 > 80% → 加内存
- 打开文件慢，开机慢 → 换 SSD（如果还在用机械盘）
- 玩游戏卡 → 先降压优化，再考虑硬件

::: info "为什么 8GB 不够用了"
2016 年，8GB 是游戏本的标配，绰绰有余。2024 年，光是 Windows 11 空闲就吃 3-4GB，开个 Chrome（10 个标签 ≈ 2GB）、挂个微信+QQ（1GB+）、后台再跑几个更新服务——8GB 已经撞墙了。

这不是任何人的错。软件整体的内存需求在涨，16GB 已经成为新的底线。如果你正在选笔记本，把内存预算排在其他所有花哨功能前面。
:::

## 常见问题

**Q：电脑管家/杀毒软件有必要装吗？**

A：
- Windows 10/11 自带 Windows Defender，足够日常使用
- 装一个靠谱的就行（别装多个，它们会互相打架，把对方当病毒扫描）
- 国产"全家桶"谨慎安装，可能带来更多问题
- 推荐：火绒（国内轻量，弹窗拦截好用）、卡巴斯基（国际大厂，查杀率高）

---

**Q：SSD 需要碎片整理吗？**

A：**绝对不要。** SSD 没有磁头，不存在寻道问题。碎片整理对 SSD 只有害处——大量写入消耗寿命。Windows 10/11 会自动识别 SSD 并禁用碎片整理。

---

**Q：虚拟内存应该设多大？**

A：让系统自动管理。现在 Windows 的内存管理完全能处理好。物理内存 > 16GB 的电脑，页面文件基本不会被用到。关掉反而可能导致某些老程序崩溃。

---

**Q：清灰会把电脑搞坏吗？**

A：如果小心操作，不会。但如果你没把握，花点钱找专业人士。特别是还在保修期内，自己拆可能失去保修——先查保修条款。

---

**Q：电脑太卡，升级还是买新的？**

A：
- 老电脑（> 5 年）：建议买新的，升级性价比低
- 中等电脑（2-5 年）：加内存换 SSD，还能战 3 年
- 新电脑（< 2 年）：检查是不是软件问题，先按本章的顺序排查

---

**Q：重装系统会丢失所有软件吗？**

A：
- "重置此电脑"选择"保留我的文件"：保留个人文件，但所有软件需要重装
- 彻底重装：所有东西都要重装
- 重装前记录你需要的软件列表，以及软件的账号密码（密码管理器在[第 5 章](./05-accounts-passwords.md)讲过）

---

**Q：为什么 C 盘空间越来越少？**

A：C 盘空间减少的常见原因：
1. **系统更新**：Windows 更新保留旧版本用于回滚（可以清理）
2. **临时文件**：各种软件产生的临时文件
3. **休眠文件**：休眠功能占用的空间（可以关闭：`powercfg -h off`）
4. **页面文件**：虚拟内存（通常几 GB，不建议关）
5. **用户文件**：文档、下载、桌面默认在 C 盘

**解决方案**：定期用 WizTree 扫一遍，把用户文件夹移到其他盘。

---

**Q：开机很慢，怎么提速？**

A：按优先级排序：
1. **禁用开机启动项**（效果最大）：任务管理器 → 启动，或用 Autoruns 更彻底
2. **清理 C 盘空间**：至少留 20% 空余
3. **升级到 SSD**：如果你的系统盘还是机械硬盘，这是效果最炸裂的升级
4. **检查硬盘健康**：用 CrystalDiskInfo 看是否硬盘老化

---

**Q：浏览器占用太多内存怎么办？**

A：
1. 关闭不用的标签页
2. 禁用不常用的扩展程序
3. 启用"标签页休眠"功能（Edge 和 Chrome 都有内置的了，不需要装扩展）
4. 定期重启浏览器（别连续几周不关）
5. 考虑换用 Firefox（内存管理通常比 Chrome 收敛）

---

**Q：电脑经常蓝屏怎么办？**

A：蓝屏通常有错误代码，可以用来定位方向：
1. **内存问题**：用 Windows 内存诊断工具或 MemTest86 检测
2. **硬盘问题**：用 CrystalDiskInfo 检查健康状态
3. **驱动问题**：更新或回滚显卡/主板驱动——特别是 NVIDIA 驱动
4. **过热问题**：检查 CPU/GPU 温度
5. **软件冲突**：最近新装了什么？
6. **系统文件损坏**：跑 `sfc /scannow`

---
