## 6.1 Nuxt 文件路由系统

### 6.1.1 什么是文件路由？

在传统的 Web 开发中，你需要手动配置路由：

```javascript
// 传统方式：手动配置每个路由
const routes = [
  { path: '/', component: Home },
  { path: '/about', component: About },
  { path: '/blog', component: Blog },
]
```

但在 Nuxt 中，**文件就是路由**！你只需要在 `pages` 目录下创建文件，Nuxt 会自动生成对应的路由。

```
pages/
├── index.vue        →  /
├── about.vue        →  /about
└── blog/
    ├── index.vue    →  /blog
    └── [id].vue     →  /blog/123, /blog/456, ...
```

::: info 约定优于配置
这就是 Nuxt 的"约定优于配置"理念：遵循约定，少写配置。

文件名就是路由路径，简单直观！
:::

### 6.1.2 路由类型速查

| 文件路径 | 生成的路由 | 说明 |
|---------|-----------|------|
| `pages/index.vue` | `/` | 首页 |
| `pages/about.vue` | `/about` | 普通页面 |
| `pages/blog/index.vue` | `/blog` | 目录首页 |
| `pages/blog/[id].vue` | `/blog/:id` | 动态路由（下章详讲） |
| `pages/blog/[...slug].vue` | `/blog/*` | 捕获所有路由 |

### 6.1.3 我们的博客路由规划

```
pages/
├── index.vue           →  /           首页（个人主页）
├── about.vue           →  /about      关于页面
└── blog/
    ├── index.vue       →  /blog       文章列表 ← 本章要做的！
    └── [id].vue        →  /blog/123   文章详情 ← 下章要做的
```

## 6.5 文章卡片组件

### 6.5.1 为什么要拆分组件？

现在文章卡片的代码都写在页面里，有几个问题：

1. **代码太长**：页面文件变得臃肿
2. **无法复用**：如果首页也想显示文章，得复制代码
3. **难以维护**：修改卡片样式要在页面里找

让我们把文章卡片拆分成独立组件！

## 6.7 新手加餐：路由、列表和 API 设计

这一章你开始做"列表页"。列表页是 Web 应用里最常见的页面之一：文章列表、商品列表、消息列表、评论列表，本质都很像。

### 6.7.1 文件路由到底省掉了什么？

在传统前端项目里，你可能要手动写：

```js
{
  path: '/blog',
  component: BlogPage
}
```

Nuxt 的文件路由帮你省掉了这一步。你创建文件：

```text
app/pages/blog/index.vue
```

Nuxt 自动理解为：

```text
/blog -> app/pages/blog/index.vue
```

这就是"约定优于配置"：只要你按约定放文件，框架就自动帮你配置。

### 6.7.2 列表 API 应该返回什么？

一个文章列表 API 不应该只返回标题，还应该返回列表页需要展示的最小信息：

```json
[
  {
    "id": 1,
    "title": "我的第一篇博客",
    "summary": "这是一段摘要",
    "created_at": "2025-12-01",
    "tags": ["学习", "Nuxt"]
  }
]
```

::: tip API 设计小原则
列表页只返回列表需要的信息，详情页再返回完整内容。

如果列表 API 一次性返回所有文章正文，数据会变大，页面也会变慢。
:::

### 6.7.3 为什么要有 `id`？

标题可能重复，文章内容可能修改，但 `id` 应该稳定。前端跳转详情页时，通常使用：

```text
/blog/1
/blog/2
```

这里的 `1`、`2` 就是文章 `id`。

可以把 `id` 理解成数据库给每条数据发的身份证号。

### 6.7.4 `v-for` 为什么需要 `:key`？

写列表时经常看到：

```vue
<ArticleCard
  v-for="article in articles"
  :key="article.id"
  :article="article"
/>
```

`:key` 是 Vue 用来识别每一项的标记。没有它，Vue 在更新列表时可能不知道哪一项变了，只能更粗暴地重新渲染。

::: warning 不推荐这样写
```vue
<div v-for="article in articles" :key="article.title">
```

标题可能重复，也可能被修改。更推荐用稳定的 `id`。
:::

### 6.7.5 小练习：给列表加空状态

真实项目里列表可能为空。你可以加一个判断：

```vue
<div v-if="articles.length === 0">
  暂时还没有文章，快去写第一篇吧！
</div>

<ArticleCard
  v-else
  v-for="article in articles"
  :key="article.id"
  :article="article"
/>
```

这叫"空状态设计"，是很多新手会忽略、但真实产品一定需要的东西。
